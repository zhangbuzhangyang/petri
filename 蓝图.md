这是一份作为项目发起人/主产品经理的**全局顶层规划**。
我们将抛弃传统的“需求池 -> 原型 -> 开发 -> 测试”的水瀑流，采用\*\*“内核驱动、渐进式涌现”**的研发路径。这个项目的本质不是在写一个游戏，而是在**培育一个生态\*\*。

# 🌵 Project Westworld：全局顶层规划书

## 一、 核心哲学与铁律

在整个开发周期中，团队必须刻在脑子里的三条铁律（任何违背这三条的设计都必须被否决）：

1. **引擎高于模型**：大模型只是“试图在这个世界生存的盲人”，后端的规则引擎（Python/Wasm）是“物理法则”。物理法则拥有最终解释权和一票否决权。
2. **状态即真理**：没有写进 State（血量、位置、物品归属）里的东西，就不存在。大模型嘴上说“我有一把枪”，只要 State 里没有，开枪动作必定被引擎拦截并惩罚。
3. **极客透明度**：一切皆可查。玩家（用户）必须能下钻看到任何一个决定背后的 Prompt、RAG 检索到的记忆条目、以及引擎的拦截日志。这是这个项目最核心的爽点。

***

## 二、 六个月演进路线图

我们将项目拆解为四个里程碑（Milestone），每个里程碑都是一个可独立运行、可发到 GitHub 获得 Star 的版本。

### 🟢 Milestone 1: "The Crucible" (熔炉) —— 闭环验证期 (第 1-4 周)

**目标**：不要地图，不要美术，不要复杂 UI。只证明一件事——“规则引擎能否有效约束大模型，并产生合理的生存博弈”。

- **场景设定**：一个封闭的废弃仓库（单节点）。
- **角色**：4 个 AI Agent（如：壮汉、医生、老妇人、小偷）。
- **核心规则**：极度匮乏的资源（只有 1 瓶水、1 把刀），血量随时间衰减，死亡即出局。
- **技术交付物**：
  - `Echo-Core` 基础框架跑通。
  - 实现 **JSON 强制输出解析器**（不管大模型怎么胡言乱语，必须被正则/Pydantic 提取出 `{"intent": "attack", "target": "doctor"}`）。
  - 实现 **裁判引擎**：校验意图合法性并执行。
- **PM 验收标准**：运行 10 次，不能出现大模型凭空变出水、或者死了的人还能说话的 Bug。必须能涌现出“结盟保水”或“先下手为强”的剧情。
- **开源动作**：发一篇技术博客《别让大模型写剧本，我们用规则引擎把它关进小黑屋》，附上 CLI 运行的 GIF，收割第一波 Star。

### 🟡 Milestone 2: "The Grid" (网格) —— 空间与记忆觉醒 (第 5-10 周)

**目标**：从“密室逃脱”走向“社会模拟”。引入空间移动和长期记忆，验证 Agent 是否能记住跨时间的恩怨。

- **场景设定**：扩展为 10 个节点的小镇（酒馆、诊所、黑市、广场等）。
- **角色**：20 个具有固定职业的 Agent（铁匠、警长、妓女、商人等）。
- **新增系统**：
  - **拓扑移动系统**：Agent 移动需要消耗 Tick，不能瞬移。
  - **RAG 长期记忆系统**：引入向量数据库。每次交互后提炼记忆存入。
  - **简单经济闭环**：基础的生产-交易-消费。
- **技术交付物**：
  - 接入 ChromaDB/Milvus。
  - 实现 **异步 Tick 调度器**（不再是一步步回车，而是后台自动按时间流逝推进）。
  - 引入 **模型分级策略**：日常闲聊用本地 8B 模型，涉及利益冲突用 GPT-4o/Claude。
- **PM 验收标准**：A 在酒馆抢了 B 的钱。第二天，A 走在街上遇到 B，B 能够通过 RAG 检索到昨天的记忆，主动发起攻击或报警。记忆检索的准确率需达到 80% 以上。
- **开源动作**：开放 `echo-memory` 模块，发布“小镇运行 24 小时后的社会关系图谱截图”，引爆技术圈。

### 🟠 Milestone 3: "The Panopticon" (全视之眼) —— 极客控制台成型 (第 11-18 周)

**目标**：把复杂的后台数据，包装成极具冲击力的前端 UI。让“看戏”和“干预”变得无比顺滑。

- **场景设定**：小镇完全体，增加昼夜交替和天气系统。
- **新增系统**：
  - **上帝干预 API**：允许通过前端输入自然语言指令修改世界状态（如：降下陨石、锁死某扇门）。
  - **Wasm 安全沙箱（可选高光）**：如果 Agent 试图“制造毒药”，其配方逻辑在 Wasm 沙箱中运算验证，防止恶意 Prompt 注入破坏主进程。
- **技术交付物**：
  - 基于 React + WebSocket 的实时监控大屏（完全按照上一份原型稿的布局实现）。
  - 实现核心的 **Prompt 透视面板**（实时显示选中 Agent 的输入输出）。
- **PM 验收标准**：非技术人员打开网页，不需要看代码，仅通过看关系图谱和事件流，就能看懂小镇正在发生什么政治斗争。能够通过输入指令成功引发一次暴乱。
- **开源动作**：发布 V0.9 Beta Web 版，提供在线体验地址（限额白名单），在 X/Twitter 和 V2EX 上进行大规模传播。

### 🔴 Milestone 4: "The Awakening" (觉醒) —— 涌现深渊 (第 19-24 周)

**目标**：打磨长线演化的深度，引入心理学机制，让 Agent 产生“性格变异”。

- **场景设定**：长期运行（游戏内时间跨度几年）。
- **新增系统**：
  - **性格动态漂移**：底层的性格参数（如“暴力值”、“信任度”）不再是固定的。遭受创伤会增加暴力值，长期安居会降低。
  - **反思与价值观提纯**：每天深夜，Agent 用便宜模型对一天的碎片记忆进行“做梦式总结”，提炼出高阶价值观（如“这个世界弱肉强食，我必须不择手段”）。
  - **派系与自发规则**：Agent 之间可以不依赖系统预设，自发通过对话达成“契约”，形成黑帮、工会等组织。
- **技术交付物**：
  - 引入图数据库（Neo4j）管理复杂的派系与契约关系。
  - 性格参数影响 System Prompt 的权重控制算法。
- **PM 验收标准**：系统能稳定运行 1000 个 Tick 不崩溃。涌现出没有预设的宏大剧情（例如：长期的资源压迫导致底层 Agent 自发组织起义，推翻了警长，并自发制定了新的分配规则）。
- **开源动作**：正式发布 V1.0。在 GitHub 上线 `Eco-System` 插件文档，允许社区用 YAML 写自己的规则包（比如《三体》黑暗森林法则包），彻底引爆生态。

# 🧫 Project Petri：产品愿景与演进蓝图

### 一、 一句话定位

**“一个把 AI 关进物理牢笼，让你以上帝视角观察其失控与涌现的极客沙盒。”**
它不是传统游戏（没有预设剧本），也不是 ChatGPT 套壳（没有单向问答）。它是一个**自动化社会学实验培养皿**。

### 二、 目标用户画像

1. **硬核开发者/AI 炼丹师**：想看极端 Prompt + RAG 在多智能体博弈中到底会产生什么 Bug 或奇迹。
2. **叙事设计师/跑团主**：寻找一种能自动生成意想不到戏剧冲突的“灵感发动机”。
3. **科技区 Content Creator**：需要极其硬核、容易出圈的素材（“看！这两个 AI 为了一个不存在的苹果把世界逻辑卡崩了”）。

### 三、 核心用户体验循环

整个产品的 Hook（抓手）在于\*\*“失控感”与“洞察感”\*\*的交织：
`设定初始参数 (人设/地图)` ➡️ `点击启动` ➡️ `观察控制台日志流` ➡️ **发现意料之外的行为 (震惊/爆笑)** ➡️ `打开 Prompt 检查器寻找原因 (洞察)` ➡️ `调整规则或注入事件` ➡️ 循环。

***

### 四、 产品演进路线图

我们将产品生命周期分为四个大版本，每个版本解决一个核心产品痛点。

#### 🟢 V0.1 验证期：物理法则的胜利 (当前阶段)

- **产品目标**：向世界证明，我们能做到“AI 绝对无法破坏世界物理规则”。解决目前所有 AI Agent 框架“一说谎就无敌”的痛点。
- **核心功能**：
  - 极简沙盒（如木屋/酒馆）。
  - 黑白终端日志输出。
- **用户感知**：“卧槽，它想召唤陨石被系统无情拦截了，有意思，但看日志太累。”

#### 🟡 V0.5 沉浸期：全视之眼 (即将进入)

- **产品目标**：降低认知门槛，把枯燥的 JSON 变成极具视觉冲击力的“赛博朋克控制台”。让“看戏”本身成为一种享受。
- **核心功能**：
  - **上帝视角 UI**：左侧空间拓扑图（看小人在哪）、中间实时滚动事件流（颜色高亮动作/对话/拦截）、右侧状态面板（血条/背包）。
  - **灵魂解剖台**：一键展开某个 Agent 当前的完整 Prompt（高亮显示它回忆起了什么记忆）。
  - **时间控制台**：加速（x2/x5）、暂停。
- **用户感知**“这 UI 太酷了，像在黑客帝国里看 ANT 蚁群算法。”

#### 🟠 V1.0 觉醒期：培养皿的主宰 (产品正式发布)

- **产品目标**：从“观察者”升级为“造物主”。引入时间维度，让 Agent 拥有真正漫长的“一生”。
- **核心功能**：
  - **长期记忆系统**：Agent 会记得三天前你（上帝）降下的那场雷劈。产生跨周期的仇恨或感恩。
  - **性格动态漂移**：被背叛会多疑，一直获胜会傲慢（性格数值随经历改变，影响 Prompt）。
  - **上帝干预面板**：在 UI 上直接点击“天降瘟疫”、“剥夺某人视力”，强制修改世界状态。
- **用户感知**“我创造的铁匠因为被抢了三次，现在变成了一个反社会连环杀手，而且他记得每一次。”

#### 🔴 V2.0 涌现期：失控的生态 (开源社区驱动)

- **产品目标**：打破单人体验，让社区创造价值。引入经济学与社会学复杂网络。
- **核心功能**：
  - **模组系统**：用户可以用 JSON 写自己的地图（如赛博城市、中世纪王国）和人设包，一键导入。
  - **复杂经济网络**：Agent 不只是捡东西，可以采集原料、合成工具、建立商店、发行货币。
  - **剧本录像与回放**：将一段精彩的涌现冲突（如两个帮派的火拼）导出为可分享的网页链接或视频。
- **用户感知**“这根本不是游戏，这是一个微型人类文明模拟器。”
